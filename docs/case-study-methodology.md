# Case Study Methodology

## Purpose

This methodology defines how to write a transformation case study so it is credible, reusable, and easy to scan.

## Method Steps

### 1. Define Context

- identify the transformation theme
- define the before state
- define the target outcome
- confirm the audience

### 2. Collect Evidence

- gather metrics
- gather timelines
- gather supporting artifacts
- identify the key stakeholders involved

### 3. Write The Story

- explain the business problem
- describe the change
- show the measurable outcome
- include lessons learned
- describe the business outcome in plain language

### 4. Validate

- confirm the facts
- remove sensitive details
- ensure the narrative aligns with the evidence

### 5. Publish

- place the case in the registry
- link it to the right framework repository
- include reusable metrics and template references

## Good Case Study Traits

- clear before and after
- specific but anonymized
- outcome-focused
- evidence-backed
- reusable

## Common Mistakes

- too much background and not enough impact
- vague outcome statements
- no metric references
- no connection to the broader framework

## Method Outcome

The methodology should make every case study credible, comparable, and easy to connect to the framework repos.
